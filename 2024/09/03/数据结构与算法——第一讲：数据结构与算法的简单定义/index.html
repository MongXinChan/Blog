什么是数据结构

数据结构是⼀⻔研究⾮数值计算的程序设计问题中的操作对象，以及他们之间的关系和操作等相关问题的学科 ⼈们越来越重视“数据结构”，认为程序设计的实质上对确定的问题选择⼀种好的结构，加上设计⼀种好的算法。

数据：是描述客观事物的符号，是计算机中可以操作的对象，是能够被计算机识别，并输⼊给计算机处理的符号集合。数据不仅仅包括整形、实型等数据类型，还包括字 符及声⾳、图像、视频等⾮数值类型。

这⾥说的数据，其实就是符号，⽽且这些符号必须具备两个前提

  • 可以输⼊到计算机中。

  • 能够被计算机程序处理。

​ 对于整型、实型等数值类型，可以进⾏数值计算

​ 对于字符数据类型，就需要进⾏⾮数值的处理——这⾥常常采⽤ASCII码。⽽声⾳、图像、视频等其实是可以通过编码的⼿段变成字符数据来处理。

数据元素：是组成数据的、有⼀定意义的基本单位，在计算机中通常作为整体处理，也被称为记录。 数据项：⼀个数据元素可以由若⼲个数据项组成。

​ 数据项是数据不可分割的最⼩单位。⽐如⼈有五官，四肢——⼈有姓名，年龄，性别，家庭地址，联系电话，邮政编码

数据对象：是性质相同的数据元素的集合，是数据的⼦集。

数据结构：结构，从简⽽⾔说就是关系，⽐如分⼦结构，就是说组成分⼦的原⼦之间的排列⽅式。严格点说，结构是指各个组成部份相互搭配和排列的⽅式。在现实⽣活 中，不同数据元素之间不是独⽴的，⽽是存在特定的关系，我们将这些关系称为结构。

数据结构：是相互之间存在⼀种或多种特定关系的数据元素的集合

​ 在计算机中，数据并⾮孤⽴，杂乱⽆序的，⽽是具有内在联系的数据集合。数据元素之间存在的⼀种或多种特定关系，也就是数据的组织形式。

逻辑结构：是指数据对象中数据元素之间的相互关系。

  1. 集合结构
    集合结构中的元素除了同属于⼀个集合外，他们之间没有其他关系。各个数据元素是“平等”的.

  2. 线性结构
    线性结构中的数据之间是⼀对⼀对的关系。

  3. 树形结构
    数据结构元素之间存在⼀种⼀对多的层次关系

  4. 图形结构
    数据元素是多对多的关系

    • 将每⼀个数据元素看作⼀个结点，⽤圆圈来表示。

    • 元素之间的逻辑关系⽤结点之间的连线表示，如果这个关系是有⽅向的，那么⽤带箭头的连线表示。

物理结构

  1. 顺序存储结构

    把数据元素存放在地址连续的存储单元⾥，其数据间的逻辑关系和物理关系是⼀致的

  2. 链式存储结构

    如早茶，医院叫号——排队系统，在等待的时候，你爱在哪就在哪，你关注的是前⼀个号有没有被叫到，叫到了，下⼀个就轮到你了。 把数据元素存放到任意的存储单元⾥，这组存储单元可以是连续的，也可以是不连续的。 数据元素的存储关系并不能反映其欧机关系，因此需要⽤⼀个指针存放数据元素的地址，这样通过地址就可以找到相关联数据元素的位置。

逻辑结构是⾯向问题的，⽽物理结构就是⾯向计算机的，其基本的⽬标就是将数据及其逻辑关系存储到计算机的内存中

数据类型

​ 是指⼀组性质相同的值的集合及定义在此集合上的⼀些操作的总称。
​ 为了满⾜各种需求，⽐如房⼦有公寓，有别墅，别野，胶囊公寓等等；由于不同的需求产⽣不同的产品需求，因此，要对数据进⾏分类，分出来多种数据类型。
​ C语⾔中按照取值的不同，可分为原⼦类型，结构类型。

  • ⽐如int a，b；不仅意味着为int类型变量，也同时不能超出int的取值范围。⽆论什么计算机、计算机语⾔，⼤都会⾯临着如整数运算、实数运算、字符运算 等操作，我们都可以把它们都抽象出来。
  • 抽象是指取出事物具有普遍性的本质。它是抽象出问题的特征⽽忽略⾮本质的细节，舍对具体事物的⼀个概括。

抽象数据类型（Abstract Data Type，ADT）

​ ⼀个数学模型及其定义在该模型上的⼀组操作。抽象数据类型的定义仅仅取决于它的⼀组逻辑特性，⽽与其在计算机内部如何表现和实现⽆关。

​ 抽象数据类型不仅仅只指那些已经定义并实现的数据类型，还可以是计算机编程者在设计软件程序时⾃⼰定义的数据类型，⽐如我们编写关于计算机绘图或 者地图类的软件系统，都会⽤到坐标。也就是说，总有成对的x和y，在3D系统中还有z出现，既然这三个整型数字始终是在⼀起出现，我们就定义⼀个叫point的抽象数据类型，他有xyz三个整形变量，这样我们很⽅便地操作⼀个point数据变量，就能知道这⼀点的坐标了。
​ 抽象数据类型体现了程序设计中问题分解、抽象和信息隐藏的特性。
​ 抽象数据类型把实际⽣活中的问题分解为多个规模⼩且容易处理的问题，然后建⽴⼀个计算机能处理的数据模型，并把每个功能模块的实现细节作为⼀个独⽴的单元，从⽽使具体实现过程隐藏起来。
​ 换而言之，C++所提及到的抽象性就是抽象数据类型的一种解决方式，通过对对象的数据进行模拟而来

算法

​ 算法是解决特定问题求解步骤的描述，在计算机中表现为指令的有限序列，并且每条指令表示⼀个或多个操作。 只谈数据结构，当然是可以的，但是就跟《梁⼭伯与祝英台》不谈“祝英台”，《罗密欧与朱丽叶》不谈“罗密欧”般，数据结构和算法也是这个内涵。

定义

​ 算法是解决特定问题求解步骤的描述，在计算机中表现为指令的有限序列，并且每条指令表示⼀个或多个操作。不得不提到，即使是等差数列总的求和公式也是一种算法，就算你只写printf()也是一种算法，重要的是解决问题，而不是及其步骤。

特性

​ 五个基本特性：输⼊，输出，有穷性，确定性，可⾏性

  • 输出,输⼊：算法具有零个或多个输⼊，且⾄少有⼀个或多个输出。关于零个输入，主要是指在一些傻瓜算法中，我们只是单纯求某项，或者本身输入的条件已经给了，*我们可以不要求有输入，但是一定要有输出*，否则一顿操作猛如虎，一看操作——我想要的东西呢？
  • 有穷性：算法在执⾏有限的步骤之后，⾃动结束⽽不会出现⽆限循环，并且⼀个步骤在可接受的时间内完成。这也是显然的，我们需要一个能在机器寿命就能运算出来的算法，可喜的是机器寿命相对比较短(而我们人类的寿命也有不过八万餐的说法)，但可悲的是人类经不起等待，有穷性倒不如说是人类对效率的追求的一个前提，我们的算法是寿命中能计算出来的，而我们再优化，可以将其效率提高许多
  • 确定性：算法的每⼀步骤都具有确定的含义，不会出现⼆义性。

    不得不提到一个很让编译器出bug的片段：
    int c = 6;
    c = (c++) + (++c);
    printf(“%d”, c);
    不同的编译器拿到这个问题得到的数值都有所不同。

  • 可⾏性：算法的每⼀步都必须是可⾏的，也就是说，每⼀步都能通过执⾏有限次数完成。

算法设计的要求

  • 正确性：算法的正确性是指算法⾄少应该具有输⼊、输出和加⼯处理⽆歧义性，能正确反映问题的需求，能够得到问题的正确答案
  • 可读性：算法设计的另⼀⽬的是为了便于阅读、理解和交流
  • 健壮性：当数据输⼊不合法时，算法也能做出相关处理，⽽不是产⽣异常或者莫名其妙的结果
  • 时间效率⾼和存储量低：设计算法尽量满⾜时间效率⾼和存储量低的需求 事后统计⽅法：通过设计好的测试程序和数据，利⽤计算机计时器对不同算法编织的程序的运⾏时间进⾏⽐较，从⽽确定算法效率的⾼低
  • 事前分析估算⽅法：在计算机程序编译前，依据统计⽅法对算法进⾏估算。

​ ⼀个程序的运⾏时间，依赖于算法的好坏和问题的输⼊规模。所谓问题输⼊规模是指输⼊量的多少。 我们不关⼼编写程序的过程中所⽤的程序设计语⾔是什么，也不关⼼这些程序将跑在什么样的计算机中，我们只关⼼它所实现的算法。这样，不计那些循环索引递增 和循环终⽌条件、变量声明、打印结果等操作，最终，在分析程序的运⾏时间时，最重要的是把程序看成是独⽴于程序设计语⾔的1算法或⼀系列步骤。

​ 函数的渐进增⻓ 给定两个函数f(n)和g(n),如果存在⼀个整数N，使得对于所有的n>N,f(n)总是⽐g(n)⼤,那么，我们说f(n)的增⻓渐进快于g(n) 我们可以忽略加法常数。

​ 与最⾼次项相乘的常数并不重要。 最⾼次项的指数⼤的，函数随着n的增⻓，结果也会增⻓更快。 判断⼀个算法的效率时，函数中的常数和其他次要项常常可以忽略，⽽更应该关注主项（最⾼阶项）的阶数。 某个算法，随着n的增⼤，他会越来越优于另⼀算法，或者越来越差于另⼀个算法。

算法时间复杂度

​ 在进⾏算法分析时，语法总的执⾏次数T(n)是关于问题规模n的函数，进⽽分析T(n)随n的变化情况并确定T(n)的数量级。算法的时间复杂度，也就是算法的时间量 度，记作T(n)=O(f(n))。它表示随问题规模n的增⼤，算法执⾏时间的增⻓率和f(n)的增⻓率相同，称作算法的渐进时间复杂度，简称为时间复杂度。其中f(n)是某个 问题规模n的某个函数。
​ ⽤⼤写O()来体现算法时间复杂度的记法，我们称之为⼤O记法。
​ O(1)称作常数阶,O(n)称作线性阶,O(n^2)称作平⽅阶。
推导⼤O阶

  • ⽤常数1取代运⾏时间中所有加法常数
  • 在修改后的运⾏次数函数中，只保留最⾼阶项。
  • 如果最⾼阶项存在且其系数不是1，则去除这个项相乘的系数。得到的结果就是⼤O阶。

注意：执⾏时间恒定的算法，我们称其为具有O（1）的时间复杂度，⼜叫常数阶；不论这个常数是多少我们都记作O（1）

分析算法的复杂度，关键就是要分析循环结构的运⾏情况。

——线性阶，因为不论这个常数是多少我们都记作O（1）

——显然因为是*2的关系，我们得到的次数是($log_2n$)，而我们将这种$log_an$统称为对数阶($logn$)

——平⽅阶O（$n^2$）

——平⽅阶O（$\frac{n\times n+1}{2}$）

理解⼤O阶推导不算难，难的是对数列的⼀些相关运算，这更多的是考察你的数学知识和能⼒

$$

$$

​ 平均运⾏时间是所有情况中最有意义的，因为它是期望的运⾏时间。但是现实情况中，平均运⾏时间很难通过分析得到，⼀般都是通过运⾏⼀定数量的实验数据后估算出来 ⼀般没有特殊说明的情况下，都是指最坏时间复杂度，后面我们会在排序中提及到这一所涉及的问题.

算法空间复杂度

​ 算法的空间复杂度通过计算算法所需的存储空间实现，算法空间复杂度的计算公式记作：S(n)=O(f(n)),其中，n为问题的规模，f(n)为语句关于n所占存储空间的函数

​ ⽐如查询闰年，我们是要每次查询就计算⼀次还是先计算后利⽤布尔运算来标记是否为闰年，到需要⽤到的场景再进⾏提取，是1就为闰年，否则为平年； 前者利⽤了时间最⼤程度上保证了空间，⽽后者利⽤了空间最⼤程度换取了时间。特此点名QQ,Wechat的程序员，把大部分内存用来存放这些本地数据
